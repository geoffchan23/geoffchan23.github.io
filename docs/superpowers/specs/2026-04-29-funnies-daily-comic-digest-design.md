# Funnies — Daily Comic Digest Design

**Status:** approved (2026-04-29)
**Author:** Geoff Chan + Claude
**Surface:** new section at `/funnies/` on geoffchan23.github.io

## Goal

A daily comic digest that aggregates strips from a curated list of artists Geoff
follows. Each day with at least one new strip becomes its own dated "issue" page
on the site. Reads like a newspaper's Sunday-funnies section: minimal, image-first,
with strong attribution and a direct link to support each artist.

## Non-goals

- Email or subscriber notifications. Email handling will be wired up later by Geoff
  through a separate channel; the only thing this design exposes for that is an
  Atom/RSS feed of issues at `/funnies/feed.xml`.
- Public subscriber sign-ups. No mailing list, no double opt-in, no signup form.
- Image hosting / republication. All comic images are hotlinked directly from the
  artist's CDN — `<img src="https://...artist-cdn/...">` baked into the issue page
  at write time.
- Read-state tracking ("which strips have I seen"). Issues are pure dated archives.
- Comment threads, ratings, or any user-generated content.
- Live site-rendering work. Everything is statically generated by Jekyll on push.

## URL structure

```
/funnies/                       Index — all past issues, newest first.
/funnies/2026-04-28/            One day's issue, all strips bundled.
/funnies/feed.xml               Atom feed of issues (one entry per issue).
```

A day with zero new strips produces no page; the index simply has no entry for
that date.

## Architecture

```
GitHub Actions cron (13:00 UTC daily)
  └─ python scripts/funnies_publish.py
       ├─ reads  _data/funnies_subscriptions.yml
       ├─ for each comic where active: true
       │     ├─ fetch the feed
       │     ├─ filter entries whose pubDate (in ET) == yesterday's ET date
       │     └─ extract image URL via the comic's image_strategy
       ├─ if ≥1 strip total across all comics:
       │     └─ write _funnies/YYYY-MM-DD.md
       ├─ commit data file changes (no-op if nothing new)
       └─ git push  →  GitHub Pages auto-deploys
```

13:00 UTC = 9:00 AM ET = 6:00 AM PT = 3:00 AM HST. Comfortably past midnight in
every North American timezone. The script runs once per day.

## Storage model

The site is Jekyll on GitHub Pages. The design uses two pieces of state, both
checked into the repo:

### `_data/funnies_subscriptions.yml` — hand-curated subscription list

One entry per comic. Edited by Geoff when he wants to add or remove a comic.
Read by both the publish script (Python) and Jekyll templates (for the
"subscribed comics" section on the index).

```yaml
- id: oatmeal                              # internal slug, stable
  name: "The Oatmeal"
  artist: "Matthew Inman"
  homepage: "https://theoatmeal.com"
  feed: "https://theoatmeal.com/feed/rss"
  image_strategy: feed                     # feed | og | scrape
  image_selector: ""                       # CSS selector, only when strategy: scrape
  support:
    label: "Shop"                          # button label, e.g. "Shop", "Patreon", "Tip jar"
    url: "https://theoatmeal.com/store"
  active: true
```

### `_funnies/YYYY-MM-DD.md` — Jekyll collection, one doc per issue

Written by the publish script. Each file is a Jekyll collection document with
all the strip data in front matter. Body is empty (or one optional foreword line).

```yaml
---
layout: funnies-issue
date: 2026-04-28
strips:
  - comic_id: oatmeal
    comic_name: "The Oatmeal"
    artist: "Matthew Inman"
    title: "Finish the drawing. It is NOT a plane!"
    image_url: "https://s3.amazonaws.com/theoatmeal-img/thumbnails/finish_drawing_big.png"
    source_url: "http://theoatmeal.com/comics/finish_drawing"
    homepage: "https://theoatmeal.com"
    support:
      label: "Shop"
      url: "https://theoatmeal.com/store"
  - comic_id: monkeyuser
    comic_name: "MonkeyUser"
    artist: "MonkeyUser"
    title: "Just a button"
    image_url: "https://www.monkeyuser.com/2025/button/justabutton.png"
    source_url: "https://www.monkeyuser.com/2025/button/"
    homepage: "https://www.monkeyuser.com"
    support:
      label: "Patreon"
      url: "https://www.patreon.com/monkeyuser"
---
```

The `funnies` collection is configured in `_config.yml`:

```yaml
collections:
  funnies:
    output: true
    permalink: /funnies/:path/
```

`:path` resolves to the bare filename (`2026-04-28`), giving the URL
`/funnies/2026-04-28/`.

### Why a Jekyll collection (not `data/funnies.json`)

- Each issue gets a real generated URL for free — good for SEO, sharing, archive.
- The index becomes a one-line Liquid loop: `{% for issue in site.funnies | sort: 'date' | reverse %}` (same pattern `/comics/index.html` already uses).
- The `jekyll-feed` plugin auto-publishes `/funnies/feed.xml` from the collection.
- Single source of truth. No JSON-and-collection pair to keep in sync.

## Per-comic config & image extraction

### Image extraction strategies

The publish script tries the strategy named in the subscription's
`image_strategy` field. Each strategy is a deterministic function from
(feed entry, optional config) → image URL. If the chosen strategy fails, the
script falls forward to `og` as a last-resort safety net (since most modern sites
publish a usable `og:image` even when their feeds are bare). If all strategies
fail for a strip, the script logs a warning and skips that strip; it does not
fail the whole run.

| Strategy | What it does | When to use |
|---|---|---|
| `feed` | First `<img>` inside `<content:encoded>` or `<description>` of the entry | Feed already includes the comic image (Oatmeal, Davecontra, Kohney) |
| `og` | Fetch the entry's `<link>` URL, parse `<meta property="og:image">` | Feed has only title + link; the comic page has a proper OG card (MonkeyUser) |
| `scrape` | Fetch the entry's `<link>`, apply `image_selector` CSS to the HTML | `og:image` is missing, wrong, or not the comic itself |

### Date semantics — "did this comic publish yesterday?"

For each feed entry, the script:

1. Parses the publication date from `pubDate` (RSS), `<dc:date>` (RDF), or `<updated>` (Atom).
2. Converts it to `America/New_York`.
3. Compares the date portion (Y-M-D in ET) to *yesterday's* date in ET.

A late-Pacific-evening post (e.g., 23:30 PT on day N, which is 02:30 ET on day N+1)
lands in day N+1's ET issue. This is acceptable — within a few hours of the
artist's intended day, with no per-feed timezone tuning required.

If a feed entry has no parseable date, the script logs a warning and skips it.
None of the four starter comics have this problem.

### Hotlinking

Image URLs in the issue front matter are exactly what the script extracted —
`s3.amazonaws.com/...`, `images.squarespace-cdn.com/...`, etc. The issue layout
renders `<img src="{{ strip.image_url }}">` with no proxying. Bandwidth is the
artist's; analytics on image hits go to the artist's logs. This is the
artist-friendliest model and keeps the repo small.

If an artist later moves or removes an image, the strip in the archive will
break. That's accepted — the broken image, its caption, and the "Read on
artist's site" link still serve as a referral back to the artist.

## Workflow & scheduling

### `.github/workflows/funnies-publish.yml`

- Trigger: cron `0 13 * * *` (daily 13:00 UTC) + `workflow_dispatch` for manual runs.
- Steps:
  1. `actions/checkout@v4` with a token that can push.
  2. Set up Python 3.x.
  3. Install `feedparser`, `requests`, `beautifulsoup4`, `pyyaml`, `python-dateutil`.
  4. Run `python scripts/funnies_publish.py`.
  5. If the script wrote a new file under `_funnies/`, commit and push it.
- Author for the bot commit: `Geoff Chan <geoffchan23@gmail.com>` (matches the
  existing `data: update usage stats` convention).
- Commit message: `funnies: publish issue YYYY-MM-DD`.

### `scripts/funnies_publish.py`

Single-file Python script. Responsibilities:

1. Parse arguments. Optional `--date YYYY-MM-DD` overrides the target issue
   date; default is "yesterday in ET." Useful for backfill, testing, and
   manual `workflow_dispatch` runs targeting a specific past day.
2. Load `_data/funnies_subscriptions.yml`.
3. For each `active: true` comic:
   - Fetch the feed (HTTP GET, handle errors gracefully, log and continue on failure).
   - Parse with `feedparser`.
   - Filter entries to those whose pubDate-in-ET matches the target date.
   - For each matching entry, extract the image URL via the strategy.
4. Collate all strips across all comics into a list. Cross-artist ordering
   inside an issue follows the subscription file order.
5. If the list is empty, exit 0 with a "no strips on <date>" log line and write nothing.
6. Otherwise, write `_funnies/<YYYY-MM-DD>.md` with the front matter shape above.
7. Print a summary log of strips found / strategies used / any warnings.

The script is idempotent: re-running for the same date with the same feed
state produces the same file. Re-running after a feed has updated may produce
a different file (if a late post appeared); in that case the new file replaces
the old one — accepted, since the cron normally runs after midnight ET so this
is an edge-case for manual re-runs.

## Page surfaces

### `/funnies/2026-04-28/` — issue page

Layout file: `_layouts/funnies-issue.html`. Inherits the site's existing
`default` layout (header, font, color scheme).

Visual structure:

```
[site header]

Funnies — Tuesday, April 28, 2026
← previous issue       next issue →
─────────────────────────────────

[BIG STRIP IMAGE @ natural width, max 720px]

Finish the drawing. It is NOT a plane!
Matthew Inman · The Oatmeal
[Read on theoatmeal.com ↗]   [Support Matthew: Shop ↗]

─── divider ───

[next strip…]
```

Notes:

- No card chrome, no shadows, no rounded image frames. Just the image, a line of
  attribution, two link buttons. Newspaper feel.
- Tall comics (e.g., Davecontra at 1080×3307) render at full width and scroll.
  Wide single-panel comics also render at natural width capped by `max-width`.
- Prev/next issue navigation is computed at build time by Jekyll using the
  collection's date order. If there's no previous (oldest) or no next (newest),
  the corresponding link is omitted.
- Both buttons (`Read on …`, `Support …`) are real `<a>` tags with `target="_blank" rel="noopener"`.

### `/funnies/` — index

Layout file: `funnies/index.html`. Three sections:

```
Funnies
Daily comic digest from artists I subscribe to. New issues land
mornings ET, only on days something was posted.
[Subscribe via RSS]
─────────────────────────────────

April 28, 2026
  The Oatmeal · MonkeyUser
April 26, 2026
  The Oatmeal
April 22, 2026
  Davecontra · MonkeyUser · Kohney
…

─────────────────────────────────

Subscribed comics
  The Oatmeal — Matthew Inman [Shop ↗]
  MonkeyUser                  [Patreon ↗]
  Kohney — Neil               [Patreon ↗]
  Davecontra — Dave           [Patreon ↗]
```

Notes:

- Each issue line is a link to the issue page. The "comic names" line under each
  date is just a comma- or middot-separated list of `comic_name` values pulled
  from the issue's front matter (`{% for s in issue.strips %}{{ s.comic_name }}{% endfor %}`),
  de-duped (in case a comic somehow had multiple strips on the same day).
- The "Subscribed comics" section reads from `_data/funnies_subscriptions.yml`
  directly — every artist gets a click-through to support every time the index
  is loaded, not only on days they posted. Inactive comics (`active: false`)
  are not listed.
- "Subscribe via RSS" links to `/funnies/feed.xml`.

### `/funnies/feed.xml` — Atom feed

Hand-rolled Liquid template at `funnies/feed.xml` that iterates
`site.funnies | sort: 'date' | reverse` and emits Atom XML directly.
(`jekyll-feed` only auto-generates a feed for `posts`; arbitrary collections
need either configuration we don't want or a manual template, and the manual
template is small and gives us full control over the per-entry HTML.)

One Atom `<entry>` per issue. Each entry's `<content type="html">` contains the
same strips the issue page shows: an `<img>` per strip with hotlinked
`src`, the strip title, the artist + comic name, the "Read on artist's site"
link, and the support link. This way any RSS-to-email service Geoff plugs in
later (Buttondown, Feedrabbit, etc.) gets the full digest in one shot rather
than a teaser.

### Styling

New file: `funnies.css`, loaded by both the issue layout and the index. Matches
the rest of the site's typographic conventions:

- Libre Baskerville body, dark text on the existing site background.
- 5–5.5px border-radius if any rounded corners are used.
- Hyphenated class names (`funnies-issue-title`, `funnies-strip`, etc.).
- Buttons use existing site link styling, optionally with a small arrow glyph.

## Failure modes & resilience

| Failure | Handling |
|---|---|
| One feed 5xx / times out | Log warning, skip that comic for the day, continue with the others. |
| Feed parses but has no entries for yesterday | Normal case for slow comics. No log noise. |
| `feed` strategy returns no `<img>` | Fall forward to `og`. If `og` also fails, log and skip the strip. |
| `og` fetch returns no `og:image` meta | Log and skip the strip. |
| Hotlinked image breaks later (404 from artist's CDN) | Browser shows broken image; alt text and the "Read on …" link still work. No automated remediation. |
| All comics produce zero strips | Script writes nothing, exits 0. No commit, no page. |
| Subscription entry missing `support` block | Layout omits the support button for that strip. Read-on-artist's-site link is still rendered. |
| Workflow itself fails | GitHub Actions surfaces the failure; the existing day's content is unaffected. Manual `workflow_dispatch` re-runs the script. |

## Out of scope (future work)

These are deliberately deferred. Listed here so they don't accidentally creep
into v1.

- **Email / public subscribers.** Geoff handles personal notifications externally.
- **Multi-image strips.** Some artists post mini-comics as several images per
  entry. v1 grabs the first image only. Easy extension when needed.
- **Multiple support links per artist** (Patreon + Shop + Book). Single primary
  link in v1; schema can be widened to a list later.
- **Homepage-scraping fallback for sites with no feed.** All four starter comics
  have feeds; fragile homepage scrapers can wait until they're actually needed.
- **Read-state, "new since last visit" indicators, search.**
- **A "subscribe to a comic" form.** All curation is hand-edited by Geoff in
  `_data/funnies_subscriptions.yml`.
